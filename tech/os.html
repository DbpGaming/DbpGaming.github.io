Simple. Correct. Extensible. And Choice Driven

Write up on how i think operating systems should be designed, sort of tired of the current systems we have.

Simple. a operating system should be simple, a low line count auditable codebase, but also low memory usage.

Correct. do it right, or not at all, do not half ass implement bluetooth drivers to drive the userbase up.

Portable. every part of the OS should be a tool that can be used in conjunction with others, portable to other systems and usable with other software ecosystems.

Choice Driven. the operating system should be respectful of the users choice, and everything should be changible by an adept user.

Software that could be used for this

???

minix was one of the first microkernels and one of the inspirations for linux, ideally we'd have something more updated, but if this is our only choice it's not too bad. a secondary kernel option would be freebsd which has good driver support compared to minix.

Musl

Provides a simple libc that can be statically linked, would probably have to be modified over its linux dependance

Dash

really simple posix shell, a bit bloated on memory usage, busybox ash is better, hopefully toysh can be fixed up and replace it

Toybox

Provides simple and efficient core utils, would probably be best to fork and add plan9 features as well as strip unneeded gnu features, maybe use sbase instead and port toybox things to it.

Shell

the shell should be a mix of toysh, busybox ash, dash, oksh, and rc, keep the code as minimal as possible with the best scripting features and posix compliance.

Doas

runs a program as another user, probably should be replaced by a program added in forked toybox.

runit

super simple and fast init system. maybe use sinit to augment.

cproc

simple qbe frontend c compiler.

smdev

suckless mdev.

scron

simple crond.

LibreSSL

openssl but less bad. maybe look at bearssl

socklog

system and kernel logging services.

samurai

ninja compatible build system in pure c99.

pkgsrc

a really simple yet featured and featured package management system, supports both source and binary.
