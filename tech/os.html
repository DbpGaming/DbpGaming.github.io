Simple. Correct. Extensible. And Choice Driven

Write up on how i think operating systems should be designed, sort of tired of the current systems we have.

Simple. a operating system should be simple, a low line count auditable codebase, but also low memory usage.

Correct. do it right, or not at all, do not half ass implement bluetooth drivers to drive the userbase up.

Extensible. every part of the OS should be a tool that can be used in conjunction with others.

Choice Driven. the operating system should be respectful of the users choice, and everything should be changible by an adept user.

Software that could be used for this

Musl

Provides a simple libc that can be statically linked.

Toybox

Provides simple and efficient core utils, would probably be best to fork and add plan9 features as well as strip unneeded gnu features.

runit

super simple and fast init system.

Software that is still needed for this

Execute command as another user

doas is probably okay, id rather implement a alternative in the Toybox fork however.

C Compiler

we possibly could use SCC, or 8CC it needs to be looked into.

Shell

pdksh is an option but is not efficient enough imo, if busybox ash was standalone it would be perfect, maybe we can fix up Toybox sh.

Kernel

Linux and openbsd aren't microkernels and besides linux seems a bit too bloated, minix is outdated, gnu mach seems doesn't seem ready.

Package Manager

a simple source based package manager would be nice, portage seems bloated.
