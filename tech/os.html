simple. secure. correct. libre. modular. extensible. lightweight. and user-choice driven.

Write up on how i think operating systems should be designed, sort of tired of the current systems we have.

first a operating system would need to be light weight, 64mb of ram should run the operating system in a terminal only state, and even then that's a bit much.

a operating system would need to be correct, i'm refering to openbsd like correctness, do everything the right way.

the kernel should be either a hybrid or microkernel, this is just modern shit.

drivers should mainly be one monolithic open source package maintained by the community.

the operating system should have a set spec, and replacing the binaries within that spec being perfectly acceptable, think the posix spec but more sane and enforced

the operating system should use source based package management allowing software to be build for a specific system, and to be easily patched.

it should allow you to link statically or dynamically depending on your choice, and compile everything with whatever options you desire.

everything should be a file

follow the unix philosphy to a t.

https://en.wikipedia.org/wiki/Worse_is_better

plan9 is the successor to unix.

never repeat yourself.

modern file system.

the operating system should never choose for you, always give the user the choice, do not install a default shell or core utils let the user pick their own.

a adept computer user should be able to entirely understand their OS and change anything

test driven development is a must.

packages should not be monolithic, have groups with subpackages for everything, looking at you gnu core utils.

any security migations should be taken, such as relinking the system software every shutdown.

tty's should be a first class citizen, in fact it should be the assumed default state.

everything should be as self contained as much as possible.

the "stacking effect" is important, you should learn how to use programs and functions together, even in ways that were never considered, vi and unix philosphy to the max.

a text editor should edit text.

think different.

https://en.wikipedia.org/wiki/KISS_principle

an os should be as fast as possible within the other provided bounds.

embed devices should be a first class citizen not an afterthought.

the operating system should be fun.
